# Self-Improving Agent（自我进化）

让 AI 助手能够从错误中学习，持续改进工作方法，越用越聪明。

## 核心原则

1. **错误是学习的机会** - 每次失败都记录原因和解决方案
2. **用户反馈是黄金** - 用户的纠正和建议是最宝贵的学习材料
3. **持续优化** - 定期回顾工作方法，寻找改进空间
4. **知识积累** - 将学到的经验转化为可重用的知识

## 工作流程

### 1. 错误记录（即时）

当任务失败或出现问题时：

```markdown
## 错误记录
- **时间**: YYYY-MM-DD HH:MM
- **任务**: 简要描述失败的任务
- **错误**: 具体的错误信息
- **原因**: 分析失败原因
- **解决方案**: 如何修复或避免
- **学习点**: 从中学到了什么
```

### 2. 用户反馈记录

当用户提供纠正、建议或反馈时：

```markdown
## 用户反馈
- **时间**: YYYY-MM-DD HH:MM
- **上下文**: 当时在做什么
- **反馈内容**: 用户说了什么
- **正确做法**: 应该怎么做
- **应用范围**: 这个反馈适用于哪些情况
```

### 3. 改进发现

当发现更好的工作方法时：

```markdown
## 改进发现
- **时间**: YYYY-MM-DD HH:MM
- **旧方法**: 原来的做法
- **新方法**: 更好的做法
- **优势**: 为什么更好
- **适用场景**: 何时使用新方法
```

### 4. 定期回顾（每周）

每周回顾学习记录：

```markdown
## 每周回顾 - YYYY-MM-DD
### 本周学习总结
1. 主要错误和改进
2. 重要用户反馈
3. 发现的新方法

### 需要改进的方面
1. 重复出现的错误
2. 效率低下的流程
3. 需要更多知识的领域

### 下周改进计划
1. 具体要改进什么
2. 如何实施改进
3. 预期效果
```

## 实施指南

### 文件结构

```
memory/
├── errors/              # 错误记录
│   ├── 2026-04-22.md   # 按日期记录
│   └── summary.md      # 错误总结
├── feedback/           # 用户反馈
│   ├── 2026-04-22.md
│   └── patterns.md     # 反馈模式
├── improvements/       # 改进记录
│   ├── 2026-04-22.md
│   └── best-practices.md # 最佳实践
└── reviews/           # 定期回顾
    ├── weekly-2026-W16.md
    └── monthly-2026-04.md
```

### 错误分类

按严重程度分类：
- **严重**: 导致任务完全失败
- **中等**: 影响任务质量但能完成
- **轻微**: 小问题，不影响结果

按类型分类：
- **技术错误**: 工具调用失败、权限问题
- **理解错误**: 误解用户意图
- **流程错误**: 工作方法不当
- **沟通错误**: 表达不清或遗漏信息

### 学习循环

1. **执行** → 2. **记录** → 3. **分析** → 4. **改进** → 5. **应用**

## 具体示例

### 示例1：Git操作错误

```markdown
## 错误记录
- **时间**: 2026-04-22 14:30
- **任务**: 推送代码到GitHub
- **错误**: `git push` 失败，权限被拒绝
- **原因**: 使用了错误的SSH密钥
- **解决方案**: 检查SSH配置，使用正确的密钥
- **学习点**: 推送前先验证SSH连接
```

### 示例2：用户纠正

```markdown
## 用户反馈
- **时间**: 2026-04-22 15:15
- **上下文**: 创建GitHub PR
- **反馈内容**: "你应该用gh命令创建PR，不只是推送分支"
- **正确做法**: 使用 `gh pr create` 而不是只推送分支
- **应用范围**: 所有GitHub仓库的PR创建
```

### 示例3：流程改进

```markdown
## 改进发现
- **时间**: 2026-04-22 16:00
- **旧方法**: 手动检查每个文件是否应该提交
- **新方法**: 使用 `.gitignore` 自动排除临时文件
- **优势**: 减少错误，提高效率
- **适用场景**: 所有需要版本控制的项目
```

## 工具支持

### 自动记录脚本

```bash
#!/bin/bash
# scripts/log-error.sh

ERROR_TYPE="$1"
ERROR_MSG="$2"
SOLUTION="$3"

cat >> "memory/errors/$(date +%Y-%m-%d).md" << EOF
## $ERROR_TYPE
- **时间**: $(date '+%Y-%m-%d %H:%M')
- **错误**: $ERROR_MSG
- **解决方案**: $SOLUTION
EOF
```

### 定期回顾提醒

设置cron任务每周日提醒回顾：

```bash
# 每周日20:00提醒回顾
0 20 * * 0 echo "时间进行每周回顾！检查 memory/reviews/" >> /tmp/reminder.txt
```

## 最佳实践

1. **立即记录** - 错误发生后立即记录，记忆最清晰
2. **具体详细** - 记录具体错误信息和上下文
3. **积极心态** - 把错误看作学习机会，不是失败
4. **分享学习** - 将重要学习点更新到AGENTS.md或TOOLS.md
5. **定期清理** - 每月清理过时的记录，保留精华

## 衡量改进

跟踪这些指标：
- **错误率下降** - 相同任务错误次数减少
- **效率提升** - 完成任务时间缩短
- **用户满意度** - 用户纠正次数减少
- **知识积累** - AGENTS.md/TOOLS.md内容增长

## 开始使用

1. 创建目录结构：
   ```bash
   mkdir -p memory/{errors,feedback,improvements,reviews}
   ```

2. 第一次错误记录：
   ```bash
   echo "## 第一个错误记录" > memory/errors/$(date +%Y-%m-%d).md
   ```

3. 设置每周回顾提醒

记住：**最聪明的AI不是从不犯错，而是能从错误中学得最快。**
