# Overview

Drogon is a high-performance C++17 web application framework designed for building HTTP/HTTPS servers, WebSocket services, and RESTful APIs. It provides an asynchronous, event-driven architecture built on top of non-blocking I/O, making it suitable for handling thousands of concurrent connections with minimal resource usage. The framework includes built-in support for JSON serialization, database ORM (PostgreSQL, MySQL, SQLite3), template rendering, and middleware chaining.

Use Drogon when you need a production-grade C++ web server that requires low latency and high throughput, such as real-time data services, IoT backends, or microservices where C++ performance is critical. It's particularly well-suited for applications that need to process many concurrent requests with predictable memory usage. Do not use Drogon for simple scripting tasks or when rapid prototyping in a higher-level language would be more productive, as the compilation overhead and complexity of C++ may not justify the performance benefits for small projects.

The key design philosophy of Drogon centers around asynchronous programming with coroutines (C++20) and callback-based handlers. The framework uses a single event loop per thread model, similar to Node.js, where all I/O operations are non-blocking. Controllers are registered as singletons or per-request instances, and the framework handles thread safety through its event loop mechanism. Drogon's architecture emphasizes zero-copy operations where possible, with careful memory management to avoid allocations in hot paths.
