## Quick Start

### One-command install (recommended)

```powershell
# Run in PowerShell (Admin not required)
irm https://dl.caomengxuan666.com/install.ps1 | iex
```

### Manual install

1. Download from [Releases](https://github.com/caomengxuan666/WinuxCmd/releases)
2. Extract to any directory, then enter `bin`
3. Run `create_links.ps1` to generate command links
4. Add `bin` to PATH

```powershell
.\create_links.ps1                    # NTFS (recommended)
.\create_links.ps1 -UseSymbolicLinks  # non-NTFS filesystem
.\create_links.ps1 -Remove            # remove all links
```

## Why WinuxCmd?

### Problem

On plain Windows terminals, Linux-style commands generated by AI tools often fail directly:

```bash
netstat -ano | grep 8080
tasklist | grep chrome | awk '{print $2}'
```

### Solution

WinuxCmd provides Linux command syntax on Windows and bridges native Windows command output into Linux-style pipelines.

```bash
netstat -ano | grep 8080
ipconfig | grep -i "ipv4"
dir /b | sort | uniq -c
```

## Technical Highlights

### 1. Pipeline bridging (key differentiator)

WinuxCmd treats `|` as a first-class bridge between Windows tools and Linux-style filters.

### 2. Startup performance

In benchmark comparisons, WinuxCmd outperforms uutils in most tested commands. See detailed benchmark notes:

- [Benchmark and architecture notes](DOCS/en/overview.md)

### 3. Distribution size

Single executable with link-based command dispatch keeps install size very small.

## FAQ

### How is this different from WSL?

WSL is a full Linux subsystem. WinuxCmd is a native Windows executable that understands Linux-style command usage.

### Will this break native Windows commands?

No. Unsupported commands fall back to native Windows command handling.

### Why C++?

For performance, minimal dependencies, and direct Win32 API access.
